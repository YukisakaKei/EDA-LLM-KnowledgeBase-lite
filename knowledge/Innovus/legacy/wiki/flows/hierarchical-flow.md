# 分层流程与原型设计流程

## 概述

分层流程是处理超大规模芯片设计的关键方法论。当设计规模超过数百万实例时，平面流程因内存需求和运行时间过长而不可行。分层流程通过"分而治之"的策略，将设计分解为可管理的分区块，每个分区块可由不同的设计团队并行独立开发。

### 分层设计的三个阶段

1. **芯片规划（Chip Planning）** — 将设计分解为块级设计，进行资源、引脚、馈通路径和时序的最优规划
2. **实现（Implementation）** — 包括块级实现和顶层实现两个子阶段
3. **芯片组装（Chip Assembly）** — 将所有块级设计连接成最终芯片

### 分层设计的关键考虑因素

- 选择顶层优先（Top-down）还是底层优先（Bottom-up）方法论
- 选择基于通道（Channel-based）、无通道（Channel-less）或窄通道（Narrow Channel）设计
- 工具是否支持多层级分层设计的布局规划和引脚分配

## 分层方法论

### 顶层优先（Top-down）方法论

**特点：**
- 由芯片顶层的尺寸、形状和 I/O 引脚位置驱动块级和分区块的放置
- 块级设计的尺寸和引脚基于顶层布局规划生成
- 支持块级和顶层设计的并行实现
- 能够进行早期、高效的资源、引脚、馈通路径和时序规划

**流程阶段：**

1. **芯片规划**
   - 最优规划资源、引脚、馈通路径和时序
   - 指定分区块和黑盒
   - 为分区块分配时序预算

2. **块级和顶层实现**
   - 块级实现：使用规划阶段确定的时序、尺寸和引脚分配约束
   - 块级实现应在 `savePartition` 步骤生成的块目录中进行
   - 顶层实现：使用块级模型数据（LEF、.lib、ILM/FlexILM、功耗模型、噪声模型）

3. **芯片组装**
   - 使用 `assembleDesign` 命令将顶层和所有块级的详细信息汇总
   - 进行完整芯片提取、功耗、时序和串扰分析

### 底层优先（Bottom-up）方法论

**特点：**
- 块级设计的尺寸、形状和引脚位置驱动顶层布局规划
- 无需提前规划，但存在性能、面积、时序和功耗不够优化的风险
- 块级和顶层实现之间可能需要多次迭代

**流程阶段：**

1. **块级实现**
   - 完成布局、布线、所有 I/O、时钟树和功耗设计
   - 块级设计尺寸可通过 `floorPlan` 命令调整
   - 支持非矩形块级设计

2. **顶层实现**
   - 为每个块级设计生成物理和时序抽象模型（ILM/FlexILM）
   - 创建顶层布局规划，其中引用块级抽象

3. **芯片组装**
   - 参考分区设计章节中的芯片组装部分
   - 汇总顶层和块级的网表和布线信息

## 分层设计方法

### 基于通道的方法（Channel-based Methodology）

- 成熟稳定的方法论
- 易于集成硬化 IP
- 通常不需要馈通缓冲
- 缺点：面积和时序可能不够优化

### 无通道方法（Channel-less Methodology）

- 可节省 5-7% 的芯片面积
- 无顶层逻辑
- 要求：
  - 多扇出网络离开分区边界的仔细规划
  - 邻接分区引脚的精确对齐
  - 复杂的主-克隆（Master-Clone）配置
  - 高度定制的时钟树
  - 迭代过程
  - 硬化 IP 的引脚连接
- 需要 Verilog 模块包装器用于 LEC 验证

### 窄通道方法（Narrow Channel Methodology）

- 平衡基于通道和无通道方法的优势
- 对引脚位置和馈通路径无严格要求
- 易于集成硬化 IP
- 提供更灵活的时钟树拓扑选择
- 收敛速度快
- 相比无通道设计有轻微的面积损耗

## 分层分区流程与能力

### 核心能力

#### 1. 分层分区（Hierarchical Partitioning）

**功能：**
- 将设计逻辑和物理分解为顶层设计和各分区块
- 将所有相关设计数据推送到分区块
- 为顶层和块创建不同的目录，支持不同设计团队独立实现

**支持特性：**
- 分区块和黑盒的定义和处理
- 多种方向和非矩形形状
- 多个实例化分区
- 多层级分区
- 分区保护带（Guard-bands）

#### 2. 引脚分配（Pin Assignment）

**自动引脚分配能力：**
- 邻接（无通道）设计
- 多层级分层设计
- 主设计和克隆设计
- 芯片级 I/O 设计（Flip chip）
- 非矩形分区形状
- 基于布线的引脚分配
- 基于放置的引脚分配

**约束设置：**
- 可在设计、单个分区或特定引脚/引脚组级别设置约束
- 内置优先级规则处理冲突

**交互式引脚编辑器：**
- 支持自定义引脚放置
- 在指定区域跨层分散引脚组
- 放置非首选层的引脚
- 符合用户指定的引脚约束

**引脚检查和优化：**
- 检查已分配的引脚
- 合法化违反约束的引脚
- 对齐布线通道中的引脚

#### 3. 馈通插入（Feedthrough Insertion）

**功能：**
- `insertPtnFeedthrough` 命令在分区路径上插入馈通网络和缓冲
- 避免网络在分区区域上布线（避免与分区布线资源冲突）
- 避免绕过分区（避免更长的路由和延迟）

**应用场景：**
- 无通道设计中必需
- 基于通道的设计中可选

**高级能力：**
- 基于放置或沿布线插入馈通
- 多电源电压设计中插入正确的缓冲
- 跨邻接多实例化分区（主/克隆）优化重用缓冲
- 多扇出网络的缓冲重用

#### 4. 时序预算分配（Timing Budgeting）

**方法：**
- 基于路径的自动预算分配方法
- 在早期非优化设计上运行虚拟设计优化，获得更准确的时序预算
- 支持快速模式（跳过虚拟优化）

**目标：**
- 将芯片级时序约束正确映射到块级约束
- 实现更好的设计收敛

#### 5. 分区组装（Assembling Partitions）

**功能：**
- `assembleDesign` 命令恢复嵌套分区数据
- 首次组装后，可进行完整芯片提取、功耗、时序和串扰分析
- 支持增量组装和更新

## 高级建模技术

### FlexModel

FlexModel 是一种轻量级抽象，用于支持 GigaScale 设计。

**特点：**
- 网表可减少高达 95%
- 保留接口边界逻辑，移除内部寄存器间的逻辑
- 提高容量和加快规划阶段的运行时间
- 一个分区可定义为 FlexModel，或每个分区可有多个 FlexModel

### Interface Logic Model (ILM)

ILM 是块的结构表示，特别是块结构的子集，包括：

- I/O 时序路径上的实例
- 时钟树实例
- 影响 I/O 时序路径信号完整性的实例或网络耦合

**优势：**
- 比 BlackBox 流程更精确的分析
- 比组合 .lib 或 .cdb 方法更了解 SI
- 可以模拟块内的时钟生成器
- 更精确的时序和 SI 减少设计迭代次数

### Flexible Interface Logic Model (FlexILM)

FlexILM 是一种简化网表，其中保留接口路径上的逻辑，移除内部路径上的逻辑。

**特点：**
- 减少时序图和物理数据中的内存
- 移除的实例被替换为放置阻挡以避免与新优化逻辑的冲突
- 移除的网络路由被 RC 网格替换以改进 RC 提取相关性
- 支持单遍分层解决方案，减少设计迭代次数

### Stylus 分层数据库流程（SHDB）

SHDB 是一种完整的分层设计流程，支持：

- 多层级分层设计
- 跨层级的数据同步和版本控制
- 使用 FlexILM 进行快速迭代
- 自动 ECO 回分区块

## 关键设计考虑

### 分层设计的挑战

1. **原型设计复杂性** — 需要高级规划和建模能力
2. **分区规划** — 需要仔细的资源和时序规划
3. **顶层时序关闭** — 需要准确的块级时序模型
4. **ECO 管理** — 后期 ECO 的管理和同步

### 设计收敛策略

- 使用 FlexModel 进行早期规划和快速迭代
- 使用 psPM 模型进行快速时序估计
- 使用 ILM/FlexILM 进行准确的块级时序建模
- 使用 SHDB 进行跨层级的数据同步和版本控制

### 性能优化建议

- 选择合适的分层方法（顶层优先 vs 底层优先）
- 选择合适的通道方法（基于通道 vs 无通道 vs 窄通道）
- 充分利用并行实现能力
- 早期进行时序预算分配和验证
- 使用自动化工具进行引脚分配和馈通插入
