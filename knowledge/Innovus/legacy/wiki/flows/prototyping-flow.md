# Prototyping Flow

## 概述

Prototyping Flow 是 Innovus 分层设计流程的一个子集，专为 GigaScale 设计优化。它允许设计人员在数分钟内找到真实问题，而不是花费数天时间，从而快速获得可实现的 floorplan。

### 核心优势

- **容量**：可在并发时序和拥塞驱动模式下处理超过 1 亿个实例
- **快速周转**：渐进式收敛流程，支持：
  - 模块全局放置
  - 增量宏单元放置
  - 详细标准单元放置
  - 使用皮秒/微米模型的快速精确时序分析

### 设计流程三阶段

1. **芯片规划（Chip Planning）**：将设计分解为块级设计，进行独立实现
2. **实现（Implementation）**：包括块级实现和基于块级抽象的顶层实现
3. **芯片组装（Chip Assembly）**：将所有块级设计连接到最终芯片

## Prototyping 方法论

Innovus prototyping flow 支持多种抽象模型，在精度和运行时间之间进行权衡：

### 模型对比

| 模型 | 精度 | 运行时间 | 内存占用 | 适用场景 |
|------|------|---------|---------|---------|
| BlackBox | 低 | 最快 | 最小 | 快速原型设计 |
| SoC Architecture Info (SAI) | 中 | 中等 | 中等 | 边界触发器级建模 |
| FlexModel | 高 | 较慢 | 较大 | 精确时序分析 |

### BlackBox 模型

BlackBox 是一个内容未完全定义的模块。虽然定义良好的模块也可以定义为 BlackBox，但 BlackBox 类似于硬块，可以调整大小、重新形状和分配引脚。

**BlackBox 基础流程**：

1. 导入设计（支持部分定义或未定义的块）
2. 指定 BlackBox 或加载包含 BlackBox 信息的 floorplan 文件
3. 进行 Floorplan
4. （可选）保存设计以保存 BlackBox 信息
5. 运行放置
6. （可选）运行 earlyGlobalRoute

**BlackBox 操作**：

- **保存**：使用 `saveDesign` 命令或 File - Save Design 菜单保存 BlackBox 信息
- **重新形状**：在 `proto_design` 期间，BlackBox 可在指定的宽高比范围内重新形状以最小化重叠

### SoC Architecture Info (SAI) 模型

SAI 基于指定的架构信息自动创建网表，支持以下操作：

1. 解析部分网表和 SAI 文件以创建包含 SAI 内容的网表
2. 添加虚拟单元以模拟定义模块的大小
3. 添加虚拟触发器以模拟模块间的边界连接
4. 添加虚拟内存或真实内存以协助子芯片 floorplan
5. 根据连接文件规范添加流水线级寄存器
6. 为定义的边界连接创建 SDC 时序约束文件
7. 定义 die 大小并读入 Innovus
8. 创建所有网络组和流水线网络组
9. 处理主克隆配置

### FlexModel 模型

FlexModel 是一种轻量级抽象，用于支持 GigaScale 设计。其网表类似于 ILM 网表，保留接口边界逻辑，移除内部寄存器间的逻辑。

**特点**：

- 可将设计网表减少高达 95%
- 提高容量和加快规划阶段的运行时间
- 一个分区可定义为 FlexModel，或每个分区可有多个 FlexModel

## GigaScale 设计支持

### 容量和运行时间挑战

随着设计规模增长到数千万或数亿个实例，分层规划阶段会出现严重的容量和运行时间限制。解决这些限制需要高级建模技术和流程能力。

### 解决方案

#### FlexModel 技术

FlexModel 为分区提供轻量级抽象，可帮助：

- 减少设计网表高达 95%
- 提高容量
- 加快规划阶段的运行时间

#### Active-logic Reduction Technology (ART)

Active-logic Reduction Technology 是一种在保持完整物理设计数据库在内存中的同时，激活设计中特定逻辑部分并屏蔽其他逻辑的技术。

**核心特性**：

- **活跃逻辑视图**：仅包含活跃逻辑部分
- **应用范围**：可应用于任何时序相关命令，如时序预算或时序优化
- **性能优势**：
  - 减少运行时间
  - 降低内存使用
  - 在时序操作中，活跃逻辑视图仅包含特定操作暴露的时序路径集合
  - 应用于时序优化时，支持跨分层优化，同时保持优化完成后的完整分层视图

**应用场景**：

- 时序预算
- 时序优化
- 拥塞分析
- 其他时序相关操作

## 顶层时序收敛

### Interface Logic Model (ILM)

ILM 是块的结构表示，特别是块结构的子集，包括：

- I/O 时序路径上的实例
- 时钟树实例
- 影响 I/O 时序路径信号完整性的实例或网络耦合

**ILM 优势**：

- 比 BlackBox 流程更精确的分析
- 比组合 .lib 或 .cdb 方法更了解 SI
- 可以模拟块内的时钟生成器
- 更精确的时序和 SI 减少设计迭代次数
- 无需表征块
- 基于实际设计数据工作
- 可用于非常大设计的初始原型设计阶段
- 允许仅修改顶层数据
- 完全保留已实现的分区
- 使用原始约束文件进行顶层分析
- 无时序异常抽象

### Flexible Interface Logic Model (FlexILM)

FlexILM 是一种简化网表，其中保留接口路径上的逻辑，移除内部路径上的逻辑。

**特点**：

- 减少时序图和物理数据中的内存
- 移除的实例被替换为放置阻挡以避免与新优化逻辑的冲突
- 移除的网络路由被 RC 网格替换以改进 RC 提取相关性
- 支持单遍分层解决方案，减少设计迭代次数

**工作流程**：

1. 在顶层设计中，可以优化 FlexILM 的接口路径
2. 网表和放置变更可自动 ECO 回分区块

**应用**：

- 用于 Stylus Hierarchical Database (SHDB) 流程
- 使用 `set_module_model` 和 `commit_module_model` 命令进行芯片组装

## 分层 Floorplan 考虑事项

### 最佳实践

1. **原型设计**：尽可能提前规划
2. **分区大小**：选择最优的平面块实现大小
3. **层次匹配**：逻辑和物理层次相匹配
4. **分区优化**：平衡分区数量和大小

### 引脚分配

自动引脚分配能力支持：

- 邻接（无通道）设计
- 多层级分层设计
- 主设计和克隆设计
- 芯片级 I/O 设计（Flip chip）
- 非矩形分区形状
- 基于布线的引脚分配
- 基于放置的引脚分配

**交互式引脚编辑器**：

- 支持自定义引脚放置
- 在指定区域跨层分散引脚组
- 放置非首选层的引脚
- 支持检查分配的引脚、合法化违反引脚和对齐路由通道中的引脚

### Feedthrough 插入

`insertPtnFeedthrough` 命令在分区中插入 feedthrough 网络和缓冲器，以：

- 避免在分区区域上路由这些网络（避免与分区路由资源冲突）
- 避免绕过分区（避免更长的路由和延迟）

**特点**：

- 在无通道设计中绝对必要，在基于通道的设计中也可能有用
- 支持基于放置或沿路由插入 Feedthrough
- 支持多供电电压设计的正确缓冲器插入
- 支持在甚至并排多实例化（主/克隆）分区中最优地重用缓冲器
- 支持为多扇出网络重用缓冲器

## 相关命令

- `saveDesign`：保存设计和 BlackBox 信息
- `proto_design`：原型设计命令
- `insertPtnFeedthrough`：插入 feedthrough 网络和缓冲器
- `floorPlan`：指定 floorplan
- `savePartition`：保存分区
- `assembleDesign`：芯片组装
- `set_module_model`：设置模块模型（SHDB 流程）
- `commit_module_model`：提交模块模型（SHDB 流程）
- `restore_module_model`：恢复模块模型（SHDB 流程）
